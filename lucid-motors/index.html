Part 1

Park Distance Warning System

Park Distance Warning is a visual warning system rendered on top of live video from the vehicle’s cameras. The system shows what areas are safe for the vehicle to move into and which could pose hazards.

concept

Warning levels

The detection grid consists of 5 regional zones, themselves made up of 3 sectors. Each sector is assigned a colour relative to how close danger is to the vehicle — these colours form a continuous blend from danger to safety.

concept

Bridging 3D & 2D viewpoints

The system needed to function as an additional layer on live video, be visually comprehensible in both 2D (overhead view) and 3D (perspective view) and not interfere with the drivers ability to identify objects around the vehicle. To meet these challenges I created a visual ‘wave’ based on a parabola that expressed the California flow that defines the Lucid experience.

prototyping & motion

Elegant urgency

The waves needed to appear with authority (as a warning system should), but still feel ‘Lucid’. To achieve this I combined an explosive beginning animation (the graphic is 95% complete at 500ms) with a really long ease.

prototyping & motion

Bringing it to life

Collaborating with extrememly talented 3D artist Ray Chang we built a 3D rig that could simulate the warning system logic. Using these graphics I built protoypes to communicate the idea internally and refine the behaviour in Lucid’s test environment.

Art direction

Making it engineer proof

Stakeholders in this process included, senior automotive engineers, UX leads and C-suite. To all these parties it was essential that in the build process no element of the experience be left to interpretation. Documentation included mathematical logic for the drawing of the lines, specifications of the 3D form, animation timing/easing and many edge case scenarios.

Part 2

Surround View Monitoring System

Surround view monitoring is a system of cameras located around the vehicle that when combined create a 360° view of the vehicles surroundings. Users can navigate via touch gestures, or tap one of the camera presets to see a particular view.

concept

A multi dimensional UI

One of the key challenges was creating a UI that would make sense in both overhead and perspective views. Whilst not competing with the system level UI across the app.

concept

Conditional controls

When a user is in perspective view some viewpoints are obscured. To address this a system of additional controls is added. These enable the user to exit quickly and navigate to the next camera preset — even if they can’t see it.

3d UX

Camera movements

Camera movements can very easily become disorientating. It was vital from a safety perspective that users always be able to orientate themselves in space. Working in Cinema 4D I created a camera rig that allowed me to test different camera paths and weed out those that could cause disorientation.
